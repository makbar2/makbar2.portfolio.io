OrderRx

Project Summary

I developed a UI for a pharmacy to replace their Excel-based prescription ordering. It displayed weekly patient orders, allowed staff to email GPs with a single click, and then the user would check off received prescriptions. This streamlined workflow reduced errors and freed staff to focus on patient care.

Tech Stack

  • React
  • C# & .NET
  • Shadcn
  • JWT Authentication

Role & Contributions

As the sole developer, I designed and implemented both the entire stack.

  • Designed an intuitive dashboard that consolidates patient data entry, GP selection, and medication requests into a single form, improving efficiency and reducing errors.
  • Developed RESTful endpoints for patients, GPs, medications, users, and the join table connecting them, ensuring smooth data flow and integrity.
  • Implemented JWT-based authentication to manage secure staff access.
  • Built email automation logic to send prescription requests directly to GPs, reducing manual intervention.

Issues

Minimal APIs vs Controllers

One challenge I encountered was using minimal APIs in .NET. As the number of endpoints increased, injecting the repository into each one became repetitive and unmanageable. While controllers would have reduced boilerplate code, I chose to continue with the minimal api's as i was too far into development to redo everything.

Outcomes & Lessons Learned

This was my first full-stack project using C# and React. I had to quickly learn state management and API integration between the two different frameworks.
